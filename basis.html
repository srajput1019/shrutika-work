First-pass approval is the signal that predicts a Basis renewal.

Basis renews on evidence, workflow by workflow — an accountant only keeps a Basis agent if they still trust its output enough to just review it. So the renewal question is never "are they logging in." It's what share of agent outputs get approved on first review versus sent back for rework — per firm, per practice area, tracked over time. A climbing curve is a safe renewal and an expansion target; a falling one is a churn flag weeks before the renewal call. This is that signal, made operable for a first RevOps hire.

Renewal-safe line: 85% first-pass —
Why not a generic pipeline metric? Seats, logins and ARR tell you a firm bought Basis — not whether they still trust it. Because Basis delivers finished work "ready for review," the honest leading indicator is whether the reviewer keeps accepting that work. First-pass approval turns Basis's own review loop into a renewal forecast, and isolates which practice area is drifting — so the save play (or the expansion motion) starts weeks before the renewal call, not on it. Newest practice areas (Audit here) run lowest first-pass — that's exactly where expansion revenue, and risk, concentrate.
